# Grrab

> 웹 이미지를 고해상도로 클립하고 저장하는 Chrome 확장 프로그램  
> Chrome extension for clipping and saving web images at high resolution

Grrab is a Manifest V3 Chrome extension. Hover an image on a page and press
**⌘C** (Mac) or **Ctrl+C** (Windows/Linux) to clip it. Clipped images can be
upscaled on-device (bundled ONNX model in an offscreen document), copied to the
clipboard, and saved to your chosen destination. Saved clips appear in the side
panel, ordered by date with a source label on each card.

**Website:** [https://grrab.app/](https://grrab.app/)  
**Chrome Web Store:** item id `mmnjbmjjkcbcbjkimkndljlpdhcbfcgc`

## Features

- **Image clipping** — hover an image, press the clip shortcut (default
  **⌘C** / **Ctrl+C**; customizable in the side panel under **Clip Shortcut**)
- **On-device upscaling** — ONNX Runtime Web (`realesr-general-x4v3.onnx`) runs
  in an offscreen document; images are not uploaded for processing
- **Side panel** — saved clips in a unified list with **date dividers**
  (Today / Yesterday / date) and a **source label** per card (`platform` or
  page URL)
- **Save destinations** — Chrome **Downloads**, a **local folder** (File System
  Access API), or **Google Drive** (`Grrab_files` subfolder via `drive.file`
  scope)
- **Google sign-in** — `chrome.identity.getAuthToken` with Firestore-backed
  clip metadata sync

## Architecture

- **Chrome extension (MV3)** — `browser-extension/chromium/` (content scripts,
  service worker, side panel, offscreen upscaling, destination picker)
- **Firebase Cloud Functions** — `functions/`
- **Firebase** — Auth (Google) and Firestore (clip metadata); dev project
  `saveurl-a8593`, prod project `saveurl-prod`
- **Static site** — `docs/` (landing page and privacy policy pages served at
  grrab.app)

Build artifacts under `browser-extension/chromium/` (`manifest.json`, `config.js`,
`vendor/`) and `browser-extension/dist/` are generated by the build scripts and
are gitignored.

## Development Setup

### Prerequisites

- Node.js (for extension build scripts; no `engines` field is declared in
  `browser-extension/package.json`)
- Chrome (latest stable)
- Google account (for Firebase / OAuth testing)

### Install and dev build

```bash
cd browser-extension
npm install
npm run build:dev
```

`npm install` runs `postinstall`, which also executes `build:dev`.

### Load unpacked (dev)

1. Open `chrome://extensions`
2. Enable **Developer mode**
3. Click **Load unpacked**
4. Select `browser-extension/chromium/`

See `browser-extension/BUILD.md` for OAuth key setup and environment-specific
extension IDs.

## Build

### Dev

```bash
cd browser-extension
npm run build:dev
```

Copies `manifest.dev.json` → `chromium/manifest.json` and
`config.dev.js` → `chromium/config.js`.

### Prod

```bash
cd browser-extension
npm run build:prod
```

Outputs:

- `browser-extension/dist/prod/` — unpacked prod extension
- `browser-extension/dist/grrab-prod.zip` — zip for Chrome Web Store upload

**Placeholder check:** `build-prod.js` warns (but still builds) if
`__PROD_KEY_PLACEHOLDER__` or `__PROD_OAUTH_CLIENT_ID_PLACEHOLDER__` remain in
the generated manifest. Confirm the build log has no placeholder warning before
shipping. Details in `browser-extension/BUILD.md`.

Other script: `npm run build:user-package` (see `browser-extension/package.json`).

## Privacy & Security

- **Privacy Policy (EN):** [https://grrab.app/privacy/](https://grrab.app/privacy/)
- **Privacy Policy (KR):** [https://grrab.app/privacy/kr/](https://grrab.app/privacy/kr/)
- **OAuth scopes** (`manifest.prod.json`): `openid`, `email`, `profile`,
  `https://www.googleapis.com/auth/drive.file`
- **Extension permissions:** `storage`, `downloads`, `tabs`, `windows`,
  `sidePanel`, `identity`, `clipboardWrite`, `offscreen`, `notifications`, plus
  `<all_urls>` host permission
- **Data:** clip metadata in Firestore; optional saves to the user's Downloads
  folder, a user-chosen local folder, or the user's Google Drive (`Grrab_files`)
- **Service accounts:** `credentials/service-account-dev.json` and
  `credentials/service-account-prod.json` are gitignored — restore per
  `DEPLOYMENT_INFO.md`
